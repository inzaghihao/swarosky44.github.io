Vuex

1.vuex 是什么？

vuex 是一个专门为 vue.js 应用设计的状态管理架构

2.vuex 为了解决什么问题而出现?

每个组件都有自己的状态，并将其存储在自身内部。但是在应用复杂的情况下，每个组件除了自身的状态还控制着部分其他组件的状态。随着事件流的增长，组件与组件之间的状态影响在组件树里面越来越复杂 ( 事件 ping-pong，子组件 dispatch 父组件，父组件再 broadcast )

3.vuex 如何解决共享状态问题？

每个组件内部的状态分为两类：本地状态 和 应用状态。应用状态不属于任何一个组件，任何组件都可以监听并及时响应应用状态引发的 DOM 更新。将应用状态放在同一个地方统一管理。

4.vuex 的应用状态实例 store 由什么组成？

state: 一个反应应用状态的对象

mutation: 一些用于改变状态的函数。Mutations 必须是同步的。

actions: 一些用于触发 mutations 的函数。包含流程中的异步操作，可以触发多个 mutations

5.如何访问state、mutation、actions？

通过 store.state 访问 state 对象；通过 store.actions 访问actions 函数；不能直接访问 mutation 函数，必须通过调用 actions或者 store.dispatch() 访问

6.如何使用 state 对象？

引入 store 对象，在计算属性中返回 store.state.xxx 等相应的状态部分

使用 store 对象必须遵守的一点是：不允许在组件内直接改变store.state ，使用计算属性确保了 store.state是只读的。store.state 的改变只能在actions里面发生

7.如何使用 mutations 函数？

触发 mutation hanler 不能直接调用，可以通过 store.dispatch('INCREMENT')

state 对象永远是 mutation handler 的第一个参数。额外的参数在第二位开始传递。

8.mutations 应当遵守哪些规则？

1.在创建 store 初始化 state 的所有属性

2.添加一个原本不存在的属性使用 Vue.set(obj, 'newProp', 123)

3.用常量为 mutations 命名

4.mutation handler 必须是同步的。所有的异步请求在 actions 里面处理

11.vuex 推荐的应用结构是什么样？

1.state存在于单个对象中

2.只有 mutation handler 可以改变 state

3.Mutations 必须是同步的，它的职责是改变 state

4.所有类似数据获取的异步操作应该封装在 actions 里面

对于计算属性中各个组件中可能存在部分能够提出来的公共方法，可以提出作为一个getters.js的文件
