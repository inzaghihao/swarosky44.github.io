this and new

new

关键字new用于执行构造函数；

var obj = new F()

那么obj 将成为F()的一个实例对象；

如果没有new关键字:

var obj = F()

可以视为

var obj = window.F()

如果F()中有返回值，则obj为返回值，如果没有返回值则obj为undefined，此时F()内部的this也全部指向全局环境；

new关键字的原理：

当一个函数前面添加了new关键字，那么new将接管整个函数的调用过程。首先new会创建一个空的对象，然后将函数内的this指向该空对象，完成构造之后，返回该对象（即使构造函数没有return或者return一个非对象的值时，也会自动返回之前创建的对象），使var obj中obj成为该对象的指引。

this关键字

在JS中所有函数都是运行在一个对象的内部，默认为window的全局对象之中，所以this可以理解为一个函数运行时所处的对象。

因为this指向是动态切换的，即在一个函数中this的指向总是指向函数的调用者，但是这个指向其实并不明确，如下ex：

如果此时调用o.f1()，则控制台会输出两次this的指向，第一次为o，第二次为window

这是因为f2是一个立即执行函数，可以视为window调用了该函数，所以在f1函数内部this的指向并不明确。

所以在开发中我们需要在一些地方明确一下我们的this的指向问题。

call方法

obj.call(me,arg1,arg2...argn)将obj内的this全部固定指向到me对象上，第一个参数后所有参数全部都是调用函数时所需的参数。

apply方法

obj.call(me,args[])类似call方法，唯一不同的是随后的传递参数用数组的形式。

bind方法

同样用于绑定this到某个对象上，bind同时还可以绑定参数。
