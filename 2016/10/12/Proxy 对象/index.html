这段代码里用到一个 Proxy 对象，特地上 MDN 看了一下关于这个对象的介绍。

在对象的基础操作上我们可以有效地通过 Proxy 实现代理模式。

虽然还是 ES6 的规范(未必所有浏览器都已经实现了该对象)，但是它确实是一个实用性很强的对象，值得专门学习一下。

基本介绍

Proxy 是一个构造函数，用于生产 p 这样的实例。所以通过 new 操作符调用。

调用时接收两个参数：

target：对象，任意类型的对象，数组、对象、函数等等

handler：处理器集合，之后详细介绍。

eg：

Demo 中我们使用了 Proxy 对象，当我们访问 Proxy 的实例 p 的属性值，会执行我们预先注册好的一个函数 get。这其实就是 JS 提供给我们一个实现代理模式的对象。(代理模式：当我们访问一个代理对象时，其实访问的是这个对象的代理，代理做了一些处理后再将请求转发给真实的对象)

handler 中包含了我们能够代理的行为 -

handler

比较常用的几种基本操作：

  • handler.has() 判断代理对象是否拥有某个属性时触发

参数

target: 目标对象

prop: 查询的属性名

返回值 boolean 值

  • handler.get() 执行对象属性值读取时触发

访问对象上的属性 或者 原型链上的属性都会触发

返回值 可以为任何值

  • handler.set() 给对象属性赋值时触发

参数

target: 目标对象

property: 目标属性

value: 设置的新值

receiver: 最初被调用的对象，比如 obj 是 proxy 的原型链上的一个对象，obj.name = ‘🍑’。这同样会触发 proxy 上的 set 方法。
