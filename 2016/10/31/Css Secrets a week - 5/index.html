难题：条纹背景

条纹设计在日常生活中是十分常见的，但是在 web 开发中，想要实现一个条纹背景，通常我们只能依靠设计师提供图片。但这一点也不灵活，每次改动都必须依赖设计师修图。但是如果我们能够利用 CSS 去实现它，那就很不爽了。

这一章节对于 linear-gradient() 的理解十分重要，在之后许多章节中作者都有用这样的技术去实现一些以往不可能的 CSS 效果。

解决方案：linear-gradient() 函数

简介

linear-gradient() 是 09 年 Mozilla 最先提出并实现。语法经过多次改变，并最终被 W3C 组织采纳，成为 CSS3 里一枚重量级的新属性。如今 linear-gradient() 的语法如下：

参数说明

  • angle: 用角度值指定渐变方向，to <side-or-corner> 是一种简洁的写法

  • color-stop: color + 终止点位置(可选)

这里我们不对 angle 做详细讲解，主要分析 color-stop 的含义。

案例 - 横向条纹

这个元素的背景可以被解析为：

0 ~ 10% 纯红色，无渐变

10% ~ 50% 由红色渐变为蓝色

50% ~ 100% 纯蓝色，无渐变

所以我们可以得知，一个颜色过渡区域的大小是可控的，我们可以无穷压缩这片区域至无穷小，那么我们将得到我们想要的🎁

我们可以随意调控这个终点位置的值，可以得到各种各样不等宽、等宽的条纹背景。

案例 - 斜向条纹

在纵向条纹的基础上，我们更加深入一部，制作斜向条纹。不得不说，斜向条纹的学习与运用不仅仅只是我们在这一章节中看到的寡淡。之后对于其的运用有更大的一片天地。

这里单纯的设置一个角度 (30deg、45deg、60deg…) 是不会成功的，因为两个颜色实现的条纹是无法实现斜向角度的无缝连接。

这样就可以实现一个斜条纹的背景，这里需要注意的是对于 background-size 的数值发生了改变，这是因为，条纹宽度不再是水平或垂直，而是斜向 45 度，所以放大了约 1.4 倍的(根号2倍)，所以同样 background-size 也需要放大了约 1.4 倍。

Tips：

在 CSS3 中规范有明确提出下面这一点：

如果某个色标的位置值比整个列表中在它之前的色标的位置值都要小，则该色标的位置值会被设置为它前面所有色标位置值最大值。

提出这一点的意义在于，我们每次修改一个 linear-gradient() 的时候，都要修改两个数值，这里可以让我们的代码更加的 DRY，前者写上真实的数值，后者取 0 即可。
