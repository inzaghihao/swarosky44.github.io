难题：实现多重边框

Css Secrets 书中介绍到：在 css工作组在讨论是否需要提供支持多重边框的属性值，当时工作组以实际需求并不多并且可以通过 border-image 设置边框图片背景来替代真正的多重边框的理由，否决了这一项提议。

但是实际开发过程中，我们对 Css 的灵活性要求超出了他们的设想，为了实现多重边框的设计，我们不得不采用多重 dom 结构来满足开发。

本章节中作者提供两种解决方案：box-shadow & outline 来实现多重边框。

解决方案

box-shadow

关于 box-shadow 的介绍，MDN 上有详细讲解：

以逗号分割一个或多个阴影效果，同时满足设置 border-radius 时，阴影也是依附于 border 来渲染，同样会产生圆角效果。

初始值： none

适用范围：所有元素 & ::first-letter

是否适用动画：true

属性值：

inset：阴影所处边框的位置，默认值是边框外，inset 是边框内

offset-x：阴影在 X 轴上的偏移量

offset-y：阴影在 Y 轴上的偏移量

blur-radius：阴影模糊度

spread-radius：阴影缩放值

color：投影的颜色

优缺点

优点：

  • 逗号分割，支持渲染任意多层边框;

  • 依附于 border ，当存在 border-radius 值时，不会出现空白间隙，会跟随产生圆角;

缺点：

  • box-shadow 不处于盒模型之中，如果通过 box-shadow 产生的阴影或者边框可能覆盖其他元素或者被覆盖;

  • 不支持 dashed dotted ... 等形式，仅仅只能模拟 solid 形式的边框;

  • 不响应悬停或者点击事件

优缺点

优点：

  • 支持 dashed dotted 等形式;

缺点：

  • 不跟随 border-radius，即渲染的 outline 可能与 border 之间产生间隙;

  • 只能渲染一层 outline，即边框最多两层;

  • 在部分浏览器中 outline 绘制边框的形状不是长方形

结论

上述两种方案模拟出的边框都有或多或少的缺陷，面对复杂的设计时，我们可以两者组合使用。但是具体得依据实际需求。

另外值得一说的是，两者都不处于盒模型之中，在整体布局之中会有意外的 bug，box-shadow也不支持响应交互。因此，推荐两者都在 border 之内渲染，通过给元素设置足够的 padding 或者宽高来适应解决。
