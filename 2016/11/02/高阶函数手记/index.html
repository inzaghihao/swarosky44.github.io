高阶函数手记

发表于 2016-11-02   |   分类于 JavaScript

这篇关于高阶函数的学习笔记是为了之后的设计模式 & 函数式编程的一系列学习路线的一个铺垫。

不能一直满足于写写业务写写组件，学习编程原则 & 学习如何高效友好的组织代码，封装对象，提供接口。

高阶函数

所谓高阶函数的意思是能够操作函数的函数，只要满足下面两者之一的函数都可以称之为高阶函数：

函数作为参数使用

可以分离一个方法中变与不变的部分，让每个调用者都具有自己个性化的地方。

对于这一种用法其实已经普及在我们的日常开发中，也是我们抽离一块代码逻辑的普通做法。而这一切得益于 JS 中函数是一等公民这一件事。

eg:

jquery 中 ajax 的用法，就提供一个 callback 参数，而这个参数就是一个函数，因为封装 ajax 函数的时候，开发者并不知道调用者在请求之后会有什么其他操作，所以提供这样一个接口便于开发者使用，个性化。

诸如此类的 JS 提供的原生函数还有许多案例，比如 Array.prototype.map, Array.prototype.sort balabala …

AOP (面向切面编程)

第一次接触 AOP 概念还是以前写 JAVA 的时候，学习 spring 框架时候。在系统架构中，除去业务功能之外还有许多其他基础必备的功能，比如日志打印、访问统计、安全控制、异常处理之流…

AOP 将这些工作提取除去，不必要和业务代码混杂在一起，导致代码混乱，重复代码过多。

这里的案例代码相对原书 《JavaScript设计模式与开发实践》的代码有一些改动，主要是我用了 es6 但是，有点地方值得提出来。

curring (柯里化)

定义：部分求值，对于一个用于连续操作的函数，不会再每次传入参数调用的时候就立刻用于求值，而是利用闭包缓存参数，等遇见特定环境时才会进行统一求值。

函数柯里化用于满足那些需要延时计算的场景，或者参数需要复用的场景

泛化

让一个对象可以使用本身并不具备的方法，这听上去就是 call & apply 的作用，的确，call & apply 可以让对象使用自身没有的方法，同时还可以泛化 this，使得泛化具有了更广的适用性。

JS 之父曾发发布了关于泛化函数的实现

函数节流

函数节流适用于当函数被频繁调用，以致于影响性能，造成卡顿。
比如 window.onresize || mousemove 这样的事件

分时函数

载入一个页面时，如果短时间 DOM 节点过多，避免造成性能上的卡顿，应该考虑使用分时函数。

总结

其实不论是设计模式还是函数式编程都是为了服务于编程这一件事，更好的组织代码更加优雅。知道或者不知道，对于代码的书写影响非常大，好比优雅的散文 & 小学生日记流水账。
