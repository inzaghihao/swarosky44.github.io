HTML & CSS 开发规范指南

发表于 2016-03-23   |   分类于 规范指南

一直以来都和组里的同事自嘲‘我的代码写的那叫一个随心所欲，想到哪写到哪’。其实不是真的故意想这样写，一直以为都想好好看一份规范，看一份最佳实践。拖拖拉拉，今天一定要好好记一记，以后多回看回看。

Html

  • 语义化：

    • 结构性元素

      • p 表示段落。只包含内联元素，不可以包含块状元素；

      • li 表示列表项。只能出现在 ul 和 ol 之中，可以包含任何元素。

      • h1~h6 表示标题。按内容重要性使用；

      • blockquote 表示引用。不可以单纯为了浏览器默认渲染的左右缩进效果而使用该标签；

      • pre 表示一段已经格式化好的文本内容。

    • 文本元素

      • a 有 href 属性表示链接，无 href 属性
        但是有 name 属性表示锚点；

      • abbr 表示缩写；

      • sub / sup 表示上标和下标；

    • 表单元素

      • button 取代 input type = "button"；

      • fieldset 和 legend 推荐用来组织表单；

      • 表单元素的 name 不能设定为 action、enctype、mehtod、novalidate、target、submit，会导致表单提交混乱。

      • label 标签添加 for 属性，增加响应区域

  • 大小写，属性值

    • 元素的标签名和属性名必须小写，属性值必须双引号

  • 行尾空格 必须删除，没有存在的意义

  • type 属性省略，link script 标签用于引入文件时的 type 属性可以省略

CSS

  • 文件引用

    • 采用 link 标签调用外部样式；

    • 不允许在页面中使用 style 块；

    • 不允许写 style 属性的行内样式；

  • 命名规范

    • 采用 “-“ 中划线作为连接符

    • 前缀采用统一规范：

  • 书写格式

    • 选择器和大括号之间一个空格

    • 冒号之后一个空格；

    • 逗号之后一个空格；

    • 所有规则之后换行；

    • 多组选择器换行；

    • 小数之前的 0 可以省略不写；

    • url() 里面的引号可以省略

    • 颜色值小写 #f00 ；

    • 遵循先布局后内容的顺序，私有属性在前标准属性在后

  • 避免低效率选择器
