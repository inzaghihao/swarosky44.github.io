this & new - Array

发表于 2016-01-24   |   分类于 JavaScript

new

关键字new用于执行构造函数；

var obj = new F()

那么 obj 将成为 F() 的一个实例对象；

如果没有 new 关键字:

var obj = F()

可以视为

var obj = window.F()

如果 F() 中有返回值，则 obj 为返回值，如果没有返回值则 obj 为 undefined，此时 F() 内部的 this 也全部指向全局环境；

new关键字的原理：

当一个函数前面添加了 new 关键字，那么 new 将接管整个函数的调用过程。首先 new 会创建一个空的对象，然后将函数内的 this 指向该空对象，完成构造之后，返回该对象（即使构造函数没有 return 或者 return 一个非对象的值时，也会自动返回之前创建的对象），使 var obj 中 obj 成为该对象的指引。

this关键字

在JS中所有函数都是运行在一个对象的内部，默认为 window 的全局对象之中，所以 this 可以理解为一个函数运行时所处的对象。

因为 this 指向是动态切换的，即在一个函数中 this 的指向总是指向函数的调用者，但是这个指向其实并不明确。eg：

如果此时调用 o.f1()，则控制台会输出两次this的指向，第一次为o，第二次为window

这是因为 f2 是一个立即执行函数，可以视为 window 调用了该函数，所以在 f1 函数内部 this 的指向并不明确。

所以在开发中我们需要在一些地方明确一下我们的this的指向问题。

call方法

obj.call(me,arg1,arg2...argn) 将 obj 内的 this 全部固定指向到 me 对象上，第一个参数后所有参数全部都是调用函数时所需的参数。

apply方法

obj.apply(me,args[]) 类似 call 方法，唯一不同的是随后的传递参数用数组的形式。

bind方法

同样用于绑定 this 到某个对象上，bind同时还可以绑定参数。
