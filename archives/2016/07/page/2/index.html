ES6 Module

概述

JS一直以来都是没有模块的概念的，一个大型项目中无法将JS代码拆分成多个小模块，没办法有效分层组织代码。除了用立即执行函数包裹代码，所有声明的变量都是全局变量，函数内的变量虽然是私有变量但是要想从外部访问函数内部的变量也只能通过闭包这样的形式才得以实现，闭包本身也存在一定性能上的问题。

因此JSer们一直在探索JS的模块化，从 require.js sea.js 到 common.js，最终演变成ES6的中的一个规范。

这里不论客户端还是服务端都推荐统一使用ES6的module，也就是我们经常写的

严格模式

ES6 的模块都是自动添加 'use static'，即使我们并没有主动在函数内部或者模块内部添加 use static

export

export 用于暴露模块内的 function、let、var、const、class 定义的变量。

注意事项：

  1. export 可以写在模块的顶部或者底部都没有问题，但是不允许在局域环境内输出，列如在函数内或者代码块里面都会报错。

  2. export 输出的是对象的引用，因此输出的内容也是动态的，也由此需要注意输出的对象的值是只读的，是不允许重新赋值的，因为会改变原来模块中对象的指引。demo：

Node中使用ES6

支持情况

Node 对 ES6 的支持分为 3 个等级：

  • shipping (已经分发并默认启用)

    • Block scoping

    • let (strict mode only)

    • const

    • function-in-blocks (strict mode only [1])

    • Classes (strict mode only)

    • Collections

    • Map

    • WeakMap

    • Set

    • WeakSet

    • Typed arrays

    • Generators

    • Binary and Octal literals

    • Object literal extensions (shorthand properties and methods)

    • Promises

    • New String methods

    • Symbols

    • Template strings

    • Arrow Functions

    • new.target [2]

    • Object.assign

    • Spread operator [2]

  • staged (已经分发，需要启用 -harmony)

    • Symbol.toStringTag

    • Array.prototype.includes （可以直接使用 polyfill 支持， 5.x 版本）

    • Rest Parameters （可以直接用 transform-es2015-parameters 转换，支持更全面， 5.x 版本）

  • in progress (开发中，不稳定或未实现，不推荐使用)

babel配置步骤

那么对于我偏偏要用 ES6 写 node 怎么办呢？我喜欢用箭头函数呀，喜欢解构呀，喜欢 let const 这样定义不同属性的变量啊。

于是 我们就需要学习使用 babel 来编译我们的 es6 语法。
